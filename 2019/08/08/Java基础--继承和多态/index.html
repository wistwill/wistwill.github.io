Java基础--继承和多态

发表于 2019-08-08 | 分类于 Java |
字数统计: 2.1k | 阅读时长 ≈ 8

继承

概述

​ 多个类中存在相同的属性和行为时，将这些内容抽取到单独一个类中，那么多个类无需在定义这些属性和行为，只要继承那个类即可

​ 单独的这个类称为父类，基类或者叫超类，多个类可以称为子类或者派生类

​ 有了继承以后，我们定义一个类的时候，可以在一个已经存在的类的基础上，还可以定义自己的新成员

继承的好处

提高了代码的复用性

抽取相同部分，进行继承，子类可以继承除私有成员的父类外的所有成员（成员变量，成员方法）

构造方法只能在构造方法中继承

提高了代码的维护性

子类随着父类改变而改变

让类与类之间产生了关系，是多态的前提

继承的弊端

弊端：

让类与类之间产生了关系，也就让类的耦合性增强了。

开发原则：高内聚，低耦合。

内聚：就是自己完成某件事情的能力

耦合：类与类的关系

继承的格式

子类名 extends 父类名

继承的特点

java中的类只支持单继承；

java中的类可以多层继承；

java继承中在成员变量的访问特点：

就近原则（局部变量->成员变量->父类的非私有成员变量）

java继承在成员方法中的特点：

就近原则

java继承中构造方法的特点:

子类所有的构造方法默认都会访问父类中空参数的构造方法

– 因为子类会继承父类中的数据，可能还会使用父类的数据，所以，子类初始化之前，一定要先完成父类数据的初始化

– 每一个构造方法的第一条默认语句都是super

– 构造方法中只能写this（…）或者supper（…）中的一个

– this（…）或者supper（…）只能放在构造方法中的第一行有效位置

super关键字

代表父类的存储空间，也可以认为是父类对象的引用

this、supper的类比

super和this的用法很像：

this:代表本类对象的引用

super:代表父类的存储空间(可以理解为代表父类对象的引用)

用法：

this super
访问成员变量： this.成员变量 super.成员变量
访问构造方法：（只能在构造方法中使用） this(…) super(…)
访问成员方法： this.成员方法() super.成员方法()

方法重写

子类中有与父类中相同声明的方法（方法名相同、形参列表、返回值类型完全一致）

作用：用于拓展功能

应用场景:

当父类原有的功能不能完全满足我们需求,我们就考虑重写父类的功能,并且基于父类原有的功能提供更强的功能

注意事项：

构造方法、set、get方法一般不重写，一般拓展新属性后添加新的set、get方法即可；

一般重写的是成员方法；

父类私有方法不能被重写（private修饰方法）；

子类重写父类方法时方法权限不能低于父类：public – protected – 默认 – private.

多态

同一对象，在不同时刻的不同形态

继承->重写->父类引用指向子类对象

多态的成员访问特点：

​ 成员变量：编译看左边（父类），执行看左边（父类）

​ 成员方法：编译看左边（父类），执行看右边（子类）

​ static修饰的成员：编译看左边（父类），执行看左边（父类）

优点：提高代码扩展性

缺点：不能调用子类特有功能

多态中的转型

类型 形式 注意 示例
向上转型 把子类对象赋值给父类类型 创建多态对象 Animals dog3=new Dog();向上转型(狗是动物)
向下转型 把父类类型对象,赋值给子类类型 想要调用子类特有的功能时,需要用到向下转型 ((Dog) dog3).setType(“可爱的”);为了能够使用setType()这个dog子类的构造方法,Animal需要向下转型为dog.

instanceof:关键字、修饰符

if（ a instanceof cat）：a属于cat向下转为猫，a属于狗，向下转为狗

快捷键

空参、有参构造：

Alt+Shift+s+c

set、get方法(用于访问私有成员变量):

Alt+Shift+s+r

有参构造：

Alt+Shift+s+o

@override(修饰被重写的方法)：

Alt+Shift+s+v
